MySQL Constraints

Constraints enforce rules on data in a table. Common constraints include:

  • PRIMARY KEY – uniquely identifies each record
  • FOREIGN KEY – ensures referential integrity
  • UNIQUE – ensures all values in a column are unique
  • NOT NULL – prevents null values
  • CHECK – ensures values satisfy a condition
  • DEFAULT – assigns a default value if none is provided
CREATE TABLE Classes (
    ClassID INT PRIMARY KEY,
    ClassName VARCHAR(50) NOT NULL,
    Teacher VARCHAR(50)
);
CREATE TABLE Students (
    StudentID INT PRIMARY KEY,
    FirstName VARCHAR(50),
    LastName VARCHAR(50),
    ClassID INT,
    FOREIGN KEY (ClassID) REFERENCES Classes(ClassID)
);

Advanced Queries

MySQL supports powerful queries using JOIN, GROUP BY, ORDER BY, and aggregate functions:

-- Select students with their class names
SELECT Students.FirstName, Students.LastName, Classes.ClassName
FROM Students
JOIN Classes ON Students.ClassID = Classes.ClassID;

-- Count students in each class
SELECT ClassID, COUNT(*) AS StudentCount
FROM Students
GROUP BY ClassID;

-- Order students by last name
SELECT * FROM Students
ORDER BY LastName ASC;

Table Relationships

MySQL uses relationships to connect tables:

  • One-to-One: Each record in Table A relates to one record in Table B
  • One-to-Many: Each record in Table A relates to multiple records in Table B
  • Many-to-Many: Use a junction table to connect multiple records from two tables
-- Junction table for Many-to-Many relationship
CREATE TABLE StudentSubjects (
    StudentID INT,
    SubjectID INT,
    PRIMARY KEY (StudentID, SubjectID),
    FOREIGN KEY (StudentID) REFERENCES Students(StudentID),
    FOREIGN KEY (SubjectID) REFERENCES Subjects(SubjectID)
);
