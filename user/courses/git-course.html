Git & GitHub Introduction

Git is a popular version control system created by Linus Torvalds in 2005, maintained by Junio Hamano.

  • Tracking code changes
  • Tracking who made changes
  • Coding collaboration

Key Git Concepts

  • Repository: Folder where Git tracks your project.
  • Clone: Copy a remote repository locally.
  • Stage: Select changes to commit.
  • Commit: Save a snapshot.
  • Branch: Work on features separately.
  • Merge: Combine branches.
  • Pull: Get latest changes.
  • Push: Send changes remotely.

Working with Git

  1. Initialize Git in a folder: git init.
  2. Git creates a hidden folder .git.
  3. Stage files: git add ..
  4. Commit files: git commit -m "message".
  5. Revert or inspect history: git log.

Why Git?

  • Over 70% of developers use Git.
  • Work collaboratively worldwide.
  • Project history visibility.
  • Ability to revert versions.

GitHub

  • Largest source code host.
  • Owned by Microsoft since 2018.
  • Use Git with GitHub for collaboration.
