Git & GitHub Introduction

Git is a popular version control system created by Linus Torvalds in 2005, maintained by Junio Hamano.

It is used for:

  • Tracking code changes
  • Tracking who made changes
  • Coding collaboration

Key Git Concepts

  • Repository: Folder where Git tracks your project and history.
  • Clone: Make a copy of a remote repository on your computer.
  • Stage: Select changes you want to save next.
  • Commit: Save a snapshot of staged changes.
  • Branch: Work on different versions or features simultaneously.
  • Merge: Combine changes from different branches.
  • Pull: Get latest changes from a remote repository.
  • Push: Send your changes to a remote repository.

Working with Git

Steps to use Git locally:

  1. Initialize Git in a folder to create a repository.
  2. Git creates a hidden folder to track changes.
  3. Modified files are staged using git add.
  4. Committed files are saved as snapshots using git commit.
  5. Git tracks history and allows reverting to previous commits.

Note: Git does not store full copies of every file but tracks changes efficiently.

Why Git?

  • Over 70% of developers use Git.
  • Developers can work together globally.
  • Full project history is visible.
  • Ability to revert to earlier versions.

Only push and pull interact with remote servers like GitHub, GitLab, or Bitbucket.

What is GitHub?

Git is different from GitHub. GitHub provides tools that use Git.

  • Largest host of source code worldwide.
  • Owned by Microsoft since 2018.
  • Focus in this tutorial will be using Git with GitHub.
