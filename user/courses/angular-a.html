AngularJS Directives

Directives are special HTML attributes (often ng-*) that extend HTML with data-binding and behavior.

ng-app

#

Declares the root of an AngularJS application. Only one auto-bootstrapped ng-app per page.

Live:

This page itself is bootstrapped via ng-app="demoApp".

<body ng-app="demoApp"> ... </body>
<script>
  angular.module('demoApp', []);
</script>

ng-model

#

Binds form controls to data (two-way):

ng-init

#

Initializes scope data inline (handy for demos).

<div ng-init="exInit={firstName:'John', lastName:'Doe'}">
  <input ng-model="exInit.firstName">
  <input ng-model="exInit.lastName">
  <p>Full Name: {{exInit.firstName}} {{exInit.lastName}}</p>
</div>

Summary

  • Directives extend HTML: ng-app, ng-model, ng-bind, ng-init.
  • Use one ng-app per page.
  • ng-model enables two-way binding.
  • ng-bind avoids interpolation flicker.
  • ng-init is for demos; prefer controllers/services for real apps.
