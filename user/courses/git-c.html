Git Stash

Stash temporarily saves your changes without committing them.

# Stash current changes
git stash

# List stashed changes
git stash list

# Apply last stashed changes
git stash apply

Click to see advanced stash commands

git stash drop stash@{0}
git stash pop
git stash branch new-feature

Git Rebase

Rebase lets you move or combine a sequence of commits to a new base commit.

# Rebase feature branch onto main
git checkout feature-branch
git rebase main

# Resolve conflicts if necessary
git rebase --continue

Git Reset

Reset can undo commits or staged changes.

# Unstage a file
git reset HEAD filename.txt

# Reset last commit but keep changes
git reset --soft HEAD~1

# Reset last commit and discard changes
git reset --hard HEAD~1

Git Best Practices

  • Commit small, logical changes often.
  • Use meaningful commit messages.
  • Use branches for features, bug fixes, and experiments.
  • Pull latest changes from remote before pushing.
  • Regularly merge or rebase branches to keep history clean.
  • Backup important branches using remote repositories.
